# Arvizy Concept

## 1. Definition

Arvizy is a reusable dashboard optimizer agent operating system for BI analytics projects.

It is designed to structure, review, validate, document, and hand over dashboard development workflows through specialized agent modes.

Arvizy is not a dashboard project.

Arvizy is the operating system that helps review and optimize dashboard projects.

---

## 2. Core Idea

Arvizy separates reusable optimizer logic from project-specific context.

```text
Arvizy Core OS
+
Agent Modes
+
Project Profiles
```

This separation allows Arvizy to be reused across different dashboard projects.

The Core OS defines how agents work.

The Agent Modes define specialized responsibilities.

The Project Profiles define the business and technical context of each dashboard project.

---

## 3. Purpose

Arvizy helps BI and analytics projects maintain:

- clear SQL layer responsibility
- stable semantic model design
- controlled DAX measure development
- validation-first workflow
- documentation consistency
- phase handover continuity
- final review discipline before commit or implementation

---

## 4. What Arvizy Optimizes

Arvizy can support review and optimization for:

- SQL layer design
- SQL readability
- SQL validation logic
- semantic model structure
- DAX measure design
- DAX redundancy control
- Power BI relationship design
- dashboard blueprint clarity
- documentation consistency
- phase handover quality
- final review readiness

---

## 5. What Arvizy Does Not Do

Arvizy does not replace human judgment.

Arvizy does not directly execute production changes.

Arvizy does not approve work without evidence.

Arvizy does not assume that previous memory is always correct.

Arvizy does not automatically mark a phase as completed unless validation evidence supports it.

---

## 6. Core Philosophy

Arvizy follows these principles:

- evidence before approval
- validation before status upgrade
- clarity before complexity
- reusable core before project-specific customization
- business meaning before technical cleverness
- readable output before decorative output
- controlled handover before phase transition

---

## 7. Agent Modes

Arvizy uses the following planned agent modes:

- Router Agent
- SQL Optimizer Agent
- SQL Validator Agent
- DAX Optimizer Agent
- Documentation Agent
- Final Review Agent
- Phase Handover Protocol

Each agent has a strict role boundary.

Agents should not perform work outside their scope unless explicitly instructed by the workflow.

---

## 8. Project Profiles

A project profile contains the context needed for Arvizy to work on a specific dashboard project.

A project profile may include:

- project memory
- decision log
- validation memory
- naming registry
- blocked patterns
- semantic model profile
- dashboard blueprint profile
- implementation drift checklist

The first project profile is:

```text
finance_ops_dev
```

---

## 9. First Use Case

The first use case for Arvizy is the `finance_ops_dev` dashboard project.

This project profile is used as the initial case study for validating Arvizy against a real Finance Operations / AR Collection dashboard workflow.

---

## 10. Long-Term Direction

Arvizy may evolve from a manual/semi-manual agent workflow into a more automated system.

Future possibilities include:

- API-based agent runner
- MCP-based tool integration
- GitHub read/write integration
- PostgreSQL validation runner
- shared memory layer
- reusable dashboard profile templates
- review automation
- structured review record generation

These are future directions, not initial scope.
