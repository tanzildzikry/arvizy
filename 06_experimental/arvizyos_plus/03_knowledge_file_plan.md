# ArvizyOS+ Knowledge File Plan

## Objective

Keep ArvizyOS+ compact, searchable, and migration-safe.

- 01_ARVIZY_CORE_OS.md
- 02_ARVIZY_ROUTER_AND_AGENT_MODES.md
- 03_ARVIZY_EVIDENCE_VALIDATION_AND_FINAL_REVIEW.md

- 09_FINANCE_OPS_DEV_VALIDATION_MEMORY.md
- 10_FINANCE_OPS_DEV_BLOCKED_PATTERNS.md

## Rules

- Do not upload obsolete drafts.
- Do not upload temporary handover files.
- Do not upload overlapping mode files unless needed.
- Prefer compact master files over many small redundant files.
- Keep Core OS reusable.
- Keep Finance Ops context project-specific.
- Keep BI Architecture Extension as one compact module first.

## Knowledge Priority

1. Core OS
2. Evidence / Validation / Final Review
3. Router and Mode Selection
4. BI Architecture Extension
5. Project Profile
6. Task Evidence

## Migration Note

ArvizyOS+ should initially be treated as an experimental agent, not as the official replacement for ArvizyOS baseline.
